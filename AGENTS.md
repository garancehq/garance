# Garance — Instructions pour agents IA

Ce fichier donne le contexte necessaire a tout agent IA travaillant sur ce repo.

## Vue d'ensemble

Garance est un BaaS souverain open source (alternative a Supabase). Le MVP comprend 8 composants dans un monorepo polyglotte (Rust, Go, TypeScript).

## Regles strictes

1. **Pas de mention IA dans les commits** — ni Co-Authored-By, ni "generated by", ni "AI-assisted"
2. **Format de commit** : gitmoji + conventional commit (`:sparkles: feat(scope): description`)
3. **Ne pas casser les tests existants** — verifier avec `cargo test` / `go test ./...` / `npm test` avant de commiter
4. **Docker requis** pour les tests d'integration (testcontainers)
5. **Ne pas modifier la structure des schemas PostgreSQL** sans mettre a jour les migrations ET les stores correspondants

## Composants et responsabilites

| Composant | Chemin | Langage | Build | Test |
|-----------|--------|---------|-------|------|
| Engine | `engine/` | Rust | `cargo build` | `cargo test` |
| Auth | `services/auth/` | Go | `go build ./...` | `go test ./... -count=1` |
| Storage | `services/storage/` | Go | `go build ./...` | `go test ./... -count=1` |
| Gateway | `services/gateway/` | Go | `go build ./...` | — |
| CLI | `cli/` | Go | `go build -o garance .` | `go test ./...` |
| Schema | `packages/schema/` | TypeScript | `npm run build` | `npm test` |
| SDK | `sdks/typescript/` | TypeScript | `npm run build` | `npm test` |
| Dashboard | `dashboard/` | TypeScript | `npm run build` | — |
| Proto | `proto/` | Protobuf | `buf generate` | `buf lint` |

## Communication inter-services

```
Client → Gateway (:8080, HTTP)
           ├─ /api/v1/*     → Engine (:5000, gRPC)
           ├─ /auth/v1/*    → Auth (:5001, gRPC)
           └─ /storage/v1/* → Storage (:5002, gRPC)
```

Les contrats gRPC sont definis dans `proto/` et generes vers :
- Go : `proto/gen/go/`
- Rust : genere a la compilation par `tonic-build` (`engine/crates/garance-engine/build.rs`)

## Schemas PostgreSQL

| Schema | Service | Tables |
|--------|---------|--------|
| `garance_auth` | Auth | users, sessions, identities, verification_tokens |
| `garance_storage` | Storage | buckets, files |
| `garance_platform` | (futur) | projects, api_keys, quotas |
| `garance_audit` | (futur) | events |
| `public` / `project_{id}` | Engine | tables utilisateur |

## Format d'erreur (toutes les API)

```json
{
  "error": {
    "code": "NOT_FOUND",
    "message": "resource not found",
    "status": 404
  }
}
```

Codes : `VALIDATION_ERROR` (400), `UNAUTHORIZED` (401), `PERMISSION_DENIED` (403), `NOT_FOUND` (404), `CONFLICT` (409), `INTERNAL_ERROR` (500).

## Dependances cles

### Rust (engine)
- axum 0.8, tonic 0.12, prost 0.13
- tokio-postgres 0.7, deadpool-postgres 0.14
- serde, uuid, chrono

### Go (auth, storage, gateway, cli)
- pgx/v5, google.golang.org/grpc
- golang-jwt/jwt/v5, golang.org/x/crypto (argon2)
- minio-go/v7 (storage), cobra (cli)

### TypeScript (schema, sdk, dashboard)
- Next.js 16, React 19, shadcn/ui, Tailwind CSS v4
- tsup, vitest
